# Documentation for `posterior.F90`

## Overview

The `posterior.F90` module is responsible for processing the raw samples generated by the MultiNest algorithm (stored in `ev.dat` and `phys_live.points`) to produce posterior distributions, statistics, and separated modal information if applicable. It calculates probability weights for each sample and can generate equally weighted posterior samples.

## Key Components

### Module `posterior`

#### `pos_samp`
*   **Type**: Subroutine
*   **Description**: This is the main routine for posterior sample processing. It reads the dead points (from `ev.dat`) and the final live points, calculates their Bayesian probability weights, and then can perform several actions:
    *   Generate a file of weighted posterior samples (`<root>.txt`).
    *   Generate a file of equally weighted posterior samples (`<root>post_equal_weights.dat`).
    *   If multimodal mode was active, it attempts to separate and characterize different modes, writing them to `<root>post_separate.dat` and `<root>stats.dat`.
    *   Calls the user-provided `dumper` function one last time with all processed samples.
*   **Parameters**:
    *   `Ztol`: (Input, `double precision`) Log-evidence threshold. Modes with `log(Z_local) < Ztol` might be ignored or handled differently.
    *   `nIter`: (Input, `integer`) Total number of iterations/replacements (globff).
    *   `root`: (Input, `character(LEN=1000)`) Root name for output files.
    *   `nLpt`: (Input, `integer`) Number of live points.
    *   `ndim`: (Input, `integer`) Dimensionality of the parameter space.
    *   `nCdim`: (Input, `integer`) Number of parameters used for clustering.
    *   `nPar`: (Input, `integer`) Total number of parameters.
    *   `multimodal`: (Input, `logical`) Flag indicating if multimodal analysis should be performed.
    *   `outfile`: (Input, `logical`) Flag to control writing of output files.
    *   `globZ`: (Input, `double precision`) Global log-evidence.
    *   `globinfo`: (Input, `double precision`) Global information content (related to error on logZ).
    *   `ic_n`: (Input, `integer`) Number of modes/nodes identified during sampling.
    *   `ic_Z`: (Input, `double precision array`) Log-evidence for each identified mode.
    *   `ic_info`: (Input, `double precision array`) Information content for each mode.
    *   `ic_reme`: (Input, `logical array`) Flags indicating if modes were re-clustered.
    *   `ic_vnow`: (Input, `double precision array`) Final prior volume associated with each mode.
    *   `ic_npt`: (Input, `integer array`) Number of points associated with each mode.
    *   `ic_nBrnch`: (Input, `integer array`) Number of branches for each mode (for tree structure).
    *   `ic_brnch`: (Input, `double precision array`) Branching information.
    *   `phyP`: (Input, `double precision array`) Physical parameters of the final live points.
    *   `l`: (Input, `double precision array`) Log-likelihoods of the final live points.
    *   `evDataAll`: (Input, `double precision array`) If `outfile` is false, this array contains the data that would have been read from `ev.dat`.
    *   `IS`: (Input, `logical`) Importance Sampling mode flag.
    *   `IS_Z`: (Input, `double precision array`) Importance Sampling log-evidence and error.
    *   `dumper`: (Input, Subroutine Interface) User-provided dumper function.
    *   `context`: (Input, `integer`) Context variable passed to `dumper`.

#### `genPoints`
*   **Type**: Recursive Subroutine
*   **Description**: This routine is likely used to traverse the tree structure of identified modes (if multimodal) and collect points belonging to each distinct mode or leaf node. It populates arrays like `consP` (constrained points, i.e., points strongly associated with a mode) and `unconsP` (unconstrained points).
*   **Parameters**:
    *   `br`: (Input, `integer`) Current branch/node being analyzed.
    *   `brNum`: (Input, `integer`) Branching number within the parent node.
    *   `nPar`: (Input, `integer`) Total number of parameters.
    *   `ic_reme`: (Input, `logical array`) Flags indicating re-clustered modes.

#### `genSepFiles`
*   **Type**: Subroutine
*   **Description**: Generates output files for separated modes. This includes writing weighted samples for each mode to `<root>post_separate.dat` and summary statistics (mean, sigma, max likelihood params, MAP params, local evidence) for each mode to `<root>stats.dat` and `<root>summary.txt`.
*   **Parameters**:
    *   `npt`: (Input, `integer`) Total number of points.
    *   `nPar`: (Input, `integer`) Total parameters.
    *   `nCls`: (Input, `integer`) Number of clusters/modes.
    *   `Ztol`: (Input, `double precision`) Log-evidence threshold.
    *   `pt`: (Input, `double precision array`) Points data.
    *   `pwt`: (Input, `double precision array`) Probability weights of points for each cluster.
    *   `nCon`: (Input, `integer array`) Number of constrained points in each cluster.
    *   `locZ`: (Input, `double precision array`) Local log-evidence for each cluster.
    *   `locInfo`: (Input, `double precision array`) Local information for each cluster.
    *   `locNpt`: (Input, `integer array`) Number of points associated with each local mode.
    *   `funit1`, `funit2`, `funit3`, `funit4`: (Input, `integer`) File units for various output files.
    *   `multimodal`: (Input, `logical`) Multimodal flag.

#### `merge`
*   **Type**: Logical Function
*   **Description**: Attempts to merge clusters/modes if they are deemed sufficiently similar (e.g., based on mean and sigma).
*   **Parameters**: Includes number of clusters, dimensionality, point data, means, sigmas, local evidences, and weights.
*   **Return Value**: `.true.` if a merge occurred, `.false.` otherwise.

#### `rGaussProp`
*   **Type**: Subroutine
*   **Description**: Calculates properties (mean, standard deviation, local evidence) of a Gaussian cluster given its points and their weights.
*   **Parameters**:
    *   `n`: (Input, `integer`) Number of points.
    *   `d`: (Input, `integer`) Dimensionality.
    *   `p`: (Input, `double precision array`) Points.
    *   `LX`: (Input, `double precision array`) Log-likelihood and log(dX) for points.
    *   `wt`: (Input, `double precision array`) Input weights.
    *   `nWt`: (Output, `double precision array`) Normalized posterior probability weights.
    *   `mean`: (Output, `double precision array`) Calculated mean.
    *   `sigma`: (Output, `double precision array`) Calculated standard deviation.
    *   `Z`: (Input/Output, `double precision`) Local evidence.


## Important Variables/Constants (Module Level)

*   `evdatp`: Allocatable array to store data read from `ev.dat`.
*   `pts`, `pts2`: Allocatable arrays for holding point coordinates and likelihoods.
*   `consP`, `unconsP`: Allocatable arrays for constrained and unconstrained points during mode separation.
*   `pwt`, `pNwt`: Allocatable arrays for probability weights of samples with respect to different modes.

## Usage Examples

The `pos_samp` subroutine is typically called internally by MultiNest (`nested.F90`) at the end of a run or when an update is triggered. End-users do not usually call these routines directly. The module's primary function is to produce the standard MultiNest output files that users then analyze.

## Dependencies and Interactions

*   **`utils1.f90`**: Uses utility functions like `LogSumExp`.
*   **`xmeans_clstr.f90`**: May use `GaussMixExpMaxLike` for refining mode properties if X-means was involved in identifying them.
*   **`nested.F90`**: The `pos_samp` routine is called from `clusteredNest` in `nested.F90`. The data processed (dead points and live points) is generated by the core nested sampling loop.
